# 学習クエスト｜高校6教科＋LEAP Basic

## これは何か

高校「歴史総合」「数学Ⅰ」「総合英語」「生物基礎」「電気回路」「数学A」と「LEAP Basic 必携英単語」対応のゲーミフィケーション・クイズ教材です。ナビゲーションは**教科選択ファーストの2階層**になっています。トップ画面には7つの教科カードが並び、進捗（クリア数・習熟度）を一目で確認できます。カードをタップすると、その教科だけのステージ一覧を表示する「教科ホーム」に移ります。スコア・SRS（間隔反復）・図鑑・称号・ボス戦などの仕組みはすべて教科横断で共通です。

歴史総合以外の6教科は、教科ホーム内で大カテゴリの下にステージが並びます。LEAP Basic は「Part × Course」を大カテゴリ、公式のWeek区切りを1ステージとして構成しています。

- **フロントエンドのみ**（サーバーサイド処理・ビルド不要）
- 外部CDN・外部フォントは使用していません（設定でBGMにYouTube動画URLを指定した場合のみ、YouTubeへの外部通信が発生します）
- スコア・コンボ・バッジ・称号・図鑑・間隔反復（SRS）による復習・ボス戦などの進捗は、ブラウザの `localStorage` に保存されます

### ローカルで確認する場合

このプロジェクトは ES Modules（`import`）で構成されているため、**`index.html` を `file://` として直接ダブルクリックで開いても動作しません**（ブラウザが `file://` オリジンからの ESM 読み込みをブロックします）。必ず簡易HTTPサーバーを立てて開いてください。

```bash
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

## PWA（ホーム画面に追加してオフラインで使う）

このアプリは PWA（Progressive Web App）として作られています。ホーム画面に追加すると、アイコンから直接起動できるアプリのような見た目・操作感になります。

- **Android Chrome**：メニューから「ホーム画面に追加」
- **iOS Safari**：共有ボタン → 「ホーム画面に追加」

一度オンラインで開いておけば、その後は電波が無い場所でも学習を続けられます（Service Worker が network-first でキャッシュしており、オンライン時は常に最新版を取得し、オフライン時だけキャッシュにフォールバックします）。

> **注意**
> - iOS でホーム画面に追加したアイコンから開く版は、Safari（ブラウザ）版とは `localStorage` の保存領域が別になります。進捗（スコア・図鑑など）はホーム画面版とSafari版で別々に管理される点に注意してください。
> - 以前の `file://` 直接開きの時代に貯めた進捗は、オリジン（保存場所の単位）が異なるため、公開URL版には引き継がれません。

### アーキテクチャ

このプロジェクトは ES Modules（`import`/`export`）だけで構成された、ビルド不要のフロントエンドです。依存関係は一方向です。

```
data/*.js  →  js/content.js  →  js/state.js  →  js/audio.js  →  js/views/*.js  →  js/main.js
```

- **データ自己登録**：各 `data/*.js` は読み込まれた瞬間、自分専用の `HQ`（`data/_registry.js` の `createRegistry()` が生成）に対して `HQ.registerUnit({...})` を呼び、自分自身を登録します。ファイル末尾で `export const units = HQ.units; export const cards = HQ.cards;` として公開し、`data/index.js` が全ファイルをまとめて1本の `units`（＋共有カード `sharedCards`）に集約します。
- **エンジン側の構築**：`js/content.js` が `data/index.js` を読み込み、起動時に `QUESTIONS`・`STAGE_ORDER`・`CARDS` を実行時に組み立てます。以降の状態管理（`js/state.js`）・画面（`js/views/*.js`）はこれらを参照するだけで、単元・教科を追加してもエンジン側の変更は不要です。
- **オフライン対応**：`sw.js`（Service Worker）が network-first 戦略でアセットをキャッシュします。オンライン時は常に最新のファイルを取得し、取得できない（オフライン）ときだけキャッシュへフォールバックするため、一度オンラインで開けば電波が無い場所でも学習を続けられます。

`index.html` が読み込む `<script>` は `<script type="module" src="js/main.js"></script>` の1本だけです。

### 階層構造（subject / group）

`HQ.registerUnit` の単元（＝1ステージ）には、任意で次の2つの見出しフィールドを付けられます。

- `subject`（教科見出し。例：`"📐 数学Ⅰ"`）… ホームの一番外側の見出し。どの教科の単元にも付けられる。
- `group`（大カテゴリ見出し。例：`"数と式"`）… `subject` の見出しの下に挟む、教科内の中分類の見出し。**省略した単元（歴史総合の3単元）は、この見出しなしで従来どおり表示される。**

現在の運用（7教科・全309ステージ・全6242問）：

- **📜 歴史総合**（`s1`〜`s3`。3ステージ・107問）… `subject:"📜 歴史総合"` のみ。`group` は使わない（大単元＝1ステージのまま。1ステージ35〜36問）。
- **📐 数学Ⅰ**（`m1s1`〜`m3s5`。15ステージ・150問）… `subject:"📐 数学Ⅰ"` ＋ `group`（`"数と式"` / `"集合と命題"` / `"2次関数"` の3種）。**小カテゴリ＝1ステージ＝ちょうど10問**という運用に統一されている。
- **🇬🇧 総合英語 Harmony**（`e1s1`〜`e2s8`。13ステージ・130問）… `subject:"🇬🇧 総合英語 Harmony"` ＋ `group`（`"英語の語順（文型）"` / `"時制"` の2種、ファイル＝大カテゴリ単位）。小カテゴリ＝1ステージ＝10問。
- **🧬 生物基礎 visual**（`b1s1`〜`b2s7`。10ステージ・100問）… `subject:"🧬 生物基礎 visual"` ＋ `group`（`"代謝とエネルギー"` / `"酵素と代謝"` / `"思考力問題"` の3種）。小カテゴリ＝1ステージ＝10問。
- **⚡ 電気回路１（実教出版）**（`ec1s1`〜`ec1s90`。90ステージ・450問）… `subject:"⚡ 電気回路１（実教出版）"` ＋ `group`（`"電気回路の要素：…"` / `"直流回路：…"` / `"交流回路：…"` / `"磁気：…"` / `"静電気：…"` など22種）。小単元＝1ステージ＝5問と、他教科より細かい単位で構成されている。
- **📐 数学A**（`ma1s1`〜`ma1s46`。46ステージ・460問）… `subject:"📐 数学A"` ＋ `group`（`"第1章 場合の数と確率：…"` / `"第2章 図形の性質：…"` / `"第3章 数学と人間の活動：…"` など章単位で7種）。小カテゴリ＝1ステージ＝10問。
- **📘 LEAP Basic 必携英単語**（`lbp1c1w1`〜`lbp5c3w9`。132ステージ・4800問）… Part 1～5を各3コース（意味・ニュアンス／フレーズ・例文／語法・Tip）で学ぶ。公式のWeek区切りと見出番号順を保ち、問題・例文メモ・Tipは独自生成。

教科選択ファースト化以降、`subject` は「教科選択画面でカードを1枚作る単位」、`group` は「教科ホーム内でサブ見出しを挟む単位」という役割です。

```
［教科選択画面］
  📜 歴史総合   （カード。タップで教科ホームへ）
  📐 数学Ⅰ     （カード。タップで教科ホームへ）

［📜 歴史総合の教科ホーム］            ［📐 数学Ⅰの教科ホーム］
  ヴェルサイユ＝ワシントン体制         数と式                          ← groupHead
  アジア諸国の独立運動                   整式の基本／展開／因数分解／実数・絶対値／平方根／一次不等式
  東アジアの民族運動と日本             集合と命題                       ← groupHead
  （groupなし・見出し無し）              集合の基本／共通部分・和集合・補集合／命題と真偽／必要十分・逆裏対偶
                                     2次関数                         ← groupHead
                                       関数の値／平方完成／頂点・軸・グラフ／平行移動／最大最小
```

（上図は2教科分の簡略図です。総合英語 Harmony・生物基礎 visual・電気回路１（実教出版）・数学Aも同じ2階層構造で、それぞれの `group` 見出しの下にステージが並びます。）

教科カードの生成ロジックは `js/views/home.js` の `renderHome()`（教科選択画面）にあり、`subjectList()`（`js/state.js`）が `STAGE_ORDER`（`order` 昇順のフラットな1本の配列）から `subject` を重複除去して順序どおりに取り出します。教科ホーム側は `js/views/subject.js` の `renderSubjectHome(subject)` にあり、`stagesOfSubject(subject)`（`js/state.js`）でその教科のステージだけに絞り込んだうえで、`group` が変化した箇所に `.groupHead` を挿入します（`STAGE_ORDER` 自体の構造は変わらないので、SRS・弱点マップ・ボス戦などの既存ロジックは影響を受けません）。CSSは教科選択画面の `.subjectCard` と、教科ホーム内の控えめな `.groupHead` を使い分けています。

### 動的化されている箇所

単元やカードを増やしても、エンジン側（`js/*.js`）を書き換える必要はありません。以下は登録された単元数・問題数から自動的に決まります。

- 教科選択画面の教科カード一覧（`subjectList()` が `STAGE_ORDER` から `subject` を重複除去して生成。教科を追加すればカードも自動で増える）
- 教科ホームのステージ一覧（`stagesOfSubject(subject)` で絞り込み。大カテゴリ見出しは `group` の変化点に挿入されるだけで、`STAGE_ORDER` 自体はフラットな1本の配列）
- ボス戦の対象ステージ、弱点マップ（教科選択画面では出さず、教科ホームでその教科内に限定して習熟度が最も低いステージを表示）
- 今日の復習（SRSのdueList）の対象範囲（教科選択画面のボタンから起動。教科横断のまま）
- 図鑑の総数表示（`CARDS.length`）・称号「学習マスター」のマスター数しきい値（全問題数と一致）
- 教科選択画面の教科カードの進捗サマリ（クリア数の分母、習熟度%）、教科ホームの「全◯問」表示・クリア数の分母

### qStatsの整合性（単元IDを変更・分割した場合の注意）

1問ごとの学習履歴（`state.qStats`）は `"単元ID-配列インデックス"`（例：`"m1s1-0"`）をキーに保存されます。単元を分割・統合してIDを変更すると、旧IDのキー（例：旧の大単元だった頃の `"m1-12"`）が保存データに残留し、現在の単元構成には存在しないゴミキーになります。

このゴミキーが `masteredCount()`（称号のしきい値判定に使う）などの集計を汚染しないよう、`js/state.js` の `load()` は起動時に `cleanupStaleQStats()` を呼び、**現在の `STAGE_ORDER`＋`QUESTIONS` に実在する `"sid-idx"` キーの集合でフィルタし、存在しないキーを削除**してから保存し直します。歴史側（`s1`〜`s3`）は単元ID・問題数を変えていないため、このタイミングでも一切削除されません（旧セーブの学習履歴・スコア・図鑑・称号はそのまま保持されます）。

## 問題の追加手順

### 既存の小カテゴリ（ステージ）に問題を追加する場合

該当する `data/mX_xxx.js`（数学）または `data/sX_xxx.js`（歴史）を開き、対象の `HQ.registerUnit({...})` 呼び出しの中の `questions` 配列の **末尾** に問題オブジェクトを追記してください。

> **重要**：既存プレイヤーの学習履歴（`qStats`）は `"単元ID-配列インデックス"` をキーにして保存されています。既存問題の並び順を変えたり、途中に挿入したりすると、他の問題の履歴がずれてしまいます。**新しい問題は必ず配列の末尾に追加してください。**
>
> 数学Ⅰ側は「小カテゴリ＝1ステージ＝ちょうど10問」という運用にしているため、追加する場合は他の小カテゴリとの整合（10問に揃えるか、あるいは運用ルールを変えるか）を検討してください。10問固定は必須の技術的制約ではなく、このプロジェクトの運用ルールです。

### 新しい小カテゴリ（ステージ）を追加する場合

同じ大カテゴリ内に小カテゴリを増やす場合は、既存単元を増やす場合と同じ要領です。対象の `data/mX_xxx.js` に `HQ.registerUnit({...})` 呼び出しを1つ追記します。

```js
HQ.registerUnit({
  id: "m1s7",              // 既存の全単元IDと重複しないIDにする（ハイフンは避ける。qStatsキーが"id-idx"のため）
  subject: "📐 数学Ⅰ",      // 教科見出し
  group: "数と式",          // 大カテゴリのサブ見出し（省略すると見出し無しで表示される）
  title: "小カテゴリのタイトル",
  desc: "小カテゴリの説明",
  order: 19,                // ホームでの表示順（昇順）。既存の最大orderより大きい数にする
  questions: [
    // 問題オブジェクトを並べる（書式は下記「問題タイプの書式」参照）。10問を目安にする。
  ],
  cards: [
    // この小カテゴリに関連する図鑑カード（省略可）
  ]
});
```

`data/index.js` 側の変更は不要です（大カテゴリのファイル自体は既に `data/index.js` から import されているため）。保存してローカルサーバーを開き直せば、ホーム画面の該当する `group` 見出しの下に新しいステージが自動的に追加されます。

### 新しい大カテゴリを追加する場合

1. `data/` フォルダに新しいファイルを作成します（例：`data/m4_zu.js`）。先頭2行に registry シムを書き、大カテゴリを構成する小カテゴリの `HQ.registerUnit({...})` 呼び出しを必要な数だけ並べ、末尾2行で `units`/`cards` を export します（既存の `data/*.js` の先頭2行・末尾2行をそのまま真似れば十分です）。

   ```js
   import { createRegistry } from "./_registry.js";
   const HQ = createRegistry();
   // ここに HQ.registerUnit({...}) を必要な数だけ
   export const units = HQ.units;
   export const cards = HQ.cards;
   ```

2. `data/index.js` に import を1行追加し、`MODULES` 配列にも加えます。

   ```js
   import * as m4 from "./m4_zu.js";
   // ...
   const MODULES = [s1, s2, s3, m1, m2, m3, /* ... */, m4];
   ```

3. `sw.js` の `ASSETS` 一覧に `data/m4_zu.js` を追加し、`CACHE_NAME` をバンプします（更新漏れは `tests/` の sw-assets 系テストが検出します）。
4. `tests/fixtures/question-digests.json` に、追加した単元の分だけ**末尾追加**で反映します（既存エントリは変更しません）。
5. `node --test tests/*.test.js` が緑であることを確認し、ローカルサーバー（`python3 -m http.server`）で開き直せば、ホーム画面に新しい大カテゴリ見出しと、その下の小カテゴリ（ステージ）が自動的に追加されます。

### 新しい教科を追加する場合

既存の7教科カード（歴史総合・数学Ⅰ・総合英語 Harmony・生物基礎 visual・電気回路１・数学A・LEAP Basic）以外の新しい教科（例：「情報Ⅰ」）を追加する場合も、単元追加の手順と同じです。

1. 新しい `subject`（例：`"🖥️ 情報Ⅰ"`。既存の7教科カードと重複しない文字列にする）を使って `data/*.js` に単元を追加する。`group` は使っても使わなくてもよい（歴史総合のように大単元＝1ステージにする場合は省略、他の6教科のように小カテゴリ単位にする場合は指定する）。
2. `order` は既存の全単元より大きい値にする（教科をホームの下側に配置するため。`order` の昇順でグループが決まるので、同じ `subject` の単元は `order` を連番にしてまとめておくとホーム表示が綺麗になります）。
3. 新しいファイルを追加した場合は、上記「新しい大カテゴリを追加する場合」の手順2〜4（`data/index.js` への import 追加／`sw.js` の `ASSETS` 更新＋`CACHE_NAME` バンプ／fixtures 末尾追加）を行います。既存ファイルに単元を追記するだけなら、これらの手順は不要です。

`subject` が同じ単元が連続していれば、ホームでは自動的に1つの見出しの下にまとめて表示され、さらにその中で `group` が同じ単元が連続していれば、大カテゴリのサブ見出しの下にまとめて表示されます（`subject`／`group` が変化した箇所にだけ見出しが挿入される仕組みのため）。称号のしきい値・図鑑の総数・「全◯問」表示なども全問題数から自動的に再計算されるので、エンジン側（`js/*.js`）の変更は不要です。

## 問題タイプの書式

問題オブジェクトは `questions` 配列の要素です。共通で `lv`（`"基礎"` または `"標準"`）、`exp`（正解後に表示する解説文）を持ちます。正解時に図鑑カードを解放したい場合は `card` に カードID（文字列、または配列で複数指定）を書きます。

任意で以下の共通フィールドが使えます（どの教科・どの問題タイプでも共通）。

- `time`：タイムアタックの制限時間（秒）を明示的に指定する。省略時はタイプごとのデフォルト（下記「制限時間」参照）が使われる。
- `hint`：ヒント文字列。指定すると問題画面に「💡 ヒント」ボタンが表示され、タップするとヒント文文が表示される（減点なし。表示は1問につき1回のみ、時間切れ・採点後は押せなくなる）。

### yon（4択）

```js
{type:"yon", lv:"基礎",
  q:"第一次世界大戦の講和のため、1919年に開かれた会議は？",
  choices:["パリ講和会議","ウィーン会議","ヤルタ会談","ワシントン会議"],
  a:0,   // 正解は choices[0]
  card:"c_example",  // 省略可
  exp:"パリ講和会議です。……"}
```

### maru（○×）

```js
{type:"maru", lv:"基礎",
  q:"ヴェルサイユ条約で、ドイツはすべての植民地を失った。",
  a:true,   // ○なら true、×なら false
  exp:"正しいです。……"}
```

### ana（穴埋め・選択式／記述モードでは自由入力になる）

```js
{type:"ana", lv:"基礎",
  q:"ドイツはヴェルサイユ条約で、巨額の（　）を課された。",
  choices:["賠償金","関税","所得税","消費税"],
  a:0,   // 正解は choices[0]
  exp:"賠償金です。……"}
```

記述（入力）モードがONのときは `choices[a]` の文字列と入力文字列を、表記ゆらぎ（全角半角・括弧・中黒・長音など）を吸収して比較します。

### kumi（組み合わせ）

```js
{type:"kumi", lv:"基礎",
  q:"次の条約と、その内容を正しく組み合わせよう。",
  pairs:[
    {l:"ヴェルサイユ条約", r:"対ドイツの講和"},
    {l:"九カ国条約",       r:"中国の主権尊重・門戸開放"}
  ],
  card:["c_a","c_b"],  // 省略可。複数カードを配列で指定できる
  exp:"……"}
```

出題時は `pairs` の `r` 側だけがシャッフルされます。

### nenpyo（年表並べ替え）

```js
{type:"nenpyo", lv:"標準",
  q:"次のできごとを、起こった年の古い順に並べよう。",
  items:[
    {t:"ヴェルサイユ条約", year:1919},
    {t:"ワシントン会議",   year:1921},
    {t:"ロカルノ条約",     year:1925}
  ],
  exp:"……"}
```

### suji（数値入力・数学向けの新タイプ）

選択肢を出さず、常にテキスト入力で答える問題です。`a` は許容する正解の配列（複数の表記を許容できる）。

```js
{type:"suji", lv:"基礎",
  q:"整式 2x²＋3x−1＋x²−5x＋4 を整理したとき、x²の係数はいくつか。",
  a:["3"],
  hint:"x²どうしをまとめる：2x²＋x²。",   // 省略可
  exp:"同じ種類の項をまとめる。x²の項は 2x²＋x²＝3x²。よってx²の係数は3。"}
```

採点は全角数字（３→3）・全角英字・マイナス記号のゆらぎ（−／－／ー等→-）を吸収したうえで文字列比較し、それでも不一致の場合は `parseFloat` による数値比較にフォールバックします（`"4"` と `"4.0"` は同一視される）。

### fill（複数穴埋め・数学向けの新タイプ）

1問に複数の入力欄を持たせたい場合に使います。`blanks` が各欄のラベル、`a` が各欄の正解（`blanks` と同じ順番・同じ長さの配列）。全欄が正解で初めてその問題が正解になります。

```js
{type:"fill", lv:"標準",
  q:"整式 3x²−x＋2 と x²＋4x−5 の和を計算し、□x²＋□x＋□ の形にせよ。",
  blanks:["x²の係数","xの係数","定数項"],
  a:["4","3","-3"],
  exp:"……"}
```

不正解時は欄ごとに正誤色がつき、フィードバックにも各欄の正解が一覧表示されます。

### junban（手順並べ替え・数学向けの新タイプ）

`nenpyo` と同じ▲▼の並べ替えUIを使いますが、正解の基準が「年号」ではなく明示的な `a`（`steps` のインデックス配列）である点が異なります。解法の手順を並べさせるような問題に使います。

```js
{type:"junban", lv:"基礎",
  q:"x²＋5x＋6 を因数分解する手順を、正しい順に並べよ。",
  steps:[
    "かけて6・たして5 になる2つの数を探す",
    "その2数は 2 と 3 とわかる",
    "(x＋2)(x＋3) と書ける"
  ],
  a:[0,1,2],   // steps[0]→steps[1]→steps[2] の順が正解
  exp:"……"}
```

### 制限時間（タイムアタック）のデフォルト

`time` を省略した場合、タイプごとに以下の制限時間（秒）が使われます。

| タイプ | 秒数 |
|---|---|
| yon / maru / ana | 15 |
| kumi / nenpyo | 25 |
| junban | 25 |
| suji / fill | 60（計算時間を確保するため長め） |

数学の4択（`yon`）問題は計算に時間がかかることが多いため、`time:30` を明示的に指定するのが移植元データでの標準的な運用です。

### 図鑑カードの追加方法

`data/sX_xxx.js`（歴史）や `data/mX_xxx.js`（数学）の `cards` 配列に、以下の形でカード定義を追加します。

```js
{id:"c_example", name:"カード名", emoji:"🎩", kind:"人物", desc:"説明文。"}
```

- `kind` は歴史側で `"人物"` / `"条約"` / `"できごと"`、数学側で `"公式"` / `"用語"` を使うのがこれまでの慣例です（任意の文字列で構いません）。
- カードを解放するには、問題オブジェクトの `card` に、そのカードの `id` を書きます（`yon`/`maru`/`ana`/`kumi`/`suji`/`fill`/`junban` など、どのタイプでも可。正解したときに解放されます。`nenpyo` には現時点で `card` タグは使われていませんが、仕組み上は同様に使えます）。
- `card` タグの `id` に対応するカード定義が存在しない場合は、正解してもカードは解放されません（無視されます）。カードを追加したら、必ずどこかの問題の `card` タグから参照するようにしてください（孤立カード防止）。

### 複数単元で共有したいカードがある場合（任意）

特定の単元に属さない共通カードは、data ファイルの末尾（`export` 文の前）で `HQ.registerCards([...])` を呼ぶことでも登録できます。これは `data/index.js` の `sharedCards` として集約され、図鑑に反映されます。通常は単元ごとの `cards` 配列に書けば十分です。

## データの初期化方法

学習の進捗・スコア・図鑑・称号などをすべてリセットしたい場合は、ブラウザの開発者コンソール（F12など）で以下を実行してください。

```js
localStorage.removeItem('rekishi-quest-v1')
```

実行後、ページを再読み込みすると初期状態から始まります。
